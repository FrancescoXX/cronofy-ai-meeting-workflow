# Cronofy AI Meeting Workflow Demo

A minimal demo application showing how to build an AI meeting workflow with Cronofy APIs.

The app uses Cronofy as the time infrastructure layer for:

- real calendar OAuth
- real availability lookup
- real calendar event creation
- automatic Google Meet generation
- Meeting Agent dispatch
- transcript and summary generation after the call

The goal is to show how a product can move from a simple meeting request to a complete workflow:

```txt
Meeting request
  ↓
Find real availability
  ↓
Book a calendar event
  ↓
Generate a Google Meet link
  ↓
Dispatch a Meeting Agent
  ↓
Record, transcribe, and summarize the meeting
```

## What this demo shows

1. A user starts with a meeting request.
2. The frontend asks the Rust backend for real calendar availability.
3. The backend calls the Cronofy Availability API.
4. The frontend displays real available slots.
5. The user selects a slot.
6. The backend creates a real Google Calendar event through Cronofy.
7. Cronofy generates a Google Meet link for the event.
8. The user dispatches a Cronofy Meeting Agent into the Google Meet.
9. The Meeting Agent joins the meeting, records it, transcribes it, and generates a summary.
10. The Cronofy dashboard shows the final transcript and summary.

## Cronofy setup

Create a Cronofy developer application.

Then configure the application credentials from:

```txt
Developer
Applications
Your Application
Credentials
```

You need:

```txt
Client ID
Client Secret
Redirect URI
```

For local development, use this redirect URI:

```txt
http://127.0.0.1:3001/oauth/callback
```

Then enable Meeting Agents:

```txt
Developer
Applications
Your Application
Meeting Agents
Enable Meeting Agents
```

The application also needs access to your Google Calendar through Cronofy OAuth.

## Environment variables

Create a `.env` file inside the `backend` folder.

```bash
cd backend
cp .env.example .env
```

Fill it with your Cronofy credentials:

```env
CRONOFY_CLIENT_ID=your_client_id
CRONOFY_CLIENT_SECRET=your_client_secret
CRONOFY_REDIRECT_URI=http://127.0.0.1:3001/oauth/callback
CRONOFY_DATA_CENTER_URL=https://api.cronofy.com
CRONOFY_APP_URL=https://app.cronofy.com
```

Optional, only needed if you also want to experiment with Scheduler Embed:

```env
CRONOFY_SCHEDULER_EMBED_PUBLIC_KEY=EMB_your_public_key
CRONOFY_SCHEDULER_EMBED_SECRET=ESK_your_embed_secret
```

Do not expose secrets in the frontend.

## Local demo flow

Start the backend:

```bash
cd backend
cargo run
```

Start the frontend:

```bash
cd frontend
npm run dev
```

Then connect Cronofy:

```txt
http://127.0.0.1:3001/oauth/start
```

After OAuth succeeds, load calendars once:

```txt
http://127.0.0.1:3001/calendars
```

Then open the app:

```txt
http://localhost:3000
```

In the frontend:

1. Click `Find availability`.
2. Select one of the available slots.
3. Click `Book selected slot`.
4. Open Google Calendar and check the created event.
5. Copy the generated Google Meet link.
6. Paste the Google Meet link into the Meeting Agent field.
7. Click `Dispatch Meeting Agent`.
8. Open the Google Meet.
9. Admit the Cronofy Meeting Agent.
10. Speak for 30 to 60 seconds.
11. Leave the meeting.
12. Open the Meeting Agent dashboard from the frontend.
13. Refresh until the state becomes `Complete`.
14. Check the transcript and summary.

Expected final result:

```txt
Calendar event created
Google Meet link generated
Meeting Agent dispatched
Meeting Agent joined the call
Recording completed
Meeting transcribed
Summary generated
```

## Notes about Meeting Agents

Meeting Agents need a real online meeting URL.

In this demo, the Google Meet URL is generated when the calendar event is created. The link is then pasted into the frontend to dispatch the Meeting Agent.

A future improvement could automatically retrieve the generated Google Meet URL from the created event and dispatch the Meeting Agent without the manual copy-paste step.

## Security notes

Do not commit `.env`.

Do not expose:

```txt
CRONOFY_CLIENT_SECRET
CRONOFY_SCHEDULER_EMBED_SECRET
OAuth access tokens
OAuth refresh tokens
```

Before making changes public, verify:

```txt
.env is ignored
.env.example contains placeholder values only
No real Cronofy secrets are committed
No access tokens are committed
No private local URLs are required for production
```
